# منصة صوت الموظف

منصة ويب داخلية لتقديم ومتابعة شكاوى الموظفين في المؤسسة.

## التشغيل

1. افتح المجلد في المتصفح أو استخدم خادم محلي:
   - **طريقة بسيطة:** انقر نقراً مزدوجاً على `index.html` لفتحها في المتصفح.
   - **خادم محلي (مفضل):** من داخل مجلد المشروع نفّذ:
     ```bash
     npx serve .
     ```
     ثم افتح الرابط المعروض (مثلاً `http://localhost:3000`).

2. **الصفحات:**
   - **الرئيسية** (`index.html`): التعريف بالمنصة، تسجيل الدخول لموظفي HR.
   - **تقديم شكوى** (`submit.html`): نموذج إرسال شكوى (مع/بدون هوية).
   - **متابعة شكوى** (`track.html`): استعلام بحالة شكوى برقم التتبع.
   - **لوحة المتابعة** (`dashboard.html`): عرض وإدارة الشكاوى (بعد تسجيل الدخول).
   - **تفاصيل الشكوى** (`complaint.html?id=XXX`): عرض وتحديث حالة شكوى.

## المميزات

- واجهة عربية كاملة مع دعم RTL.
- تقديم شكوى مع خيار المجهولية.
- تصنيف الشكاوى (مضايقة، تمييز، بيئة عمل، رواتب، إداري، أخرى).
- تتبع الحالة: مستلمة ← قيد المراجعة ← قيد المعالجة ← تمت المعالجة / مغلقة.
- لوحة متابعة مع فلترة وإحصائيات.
- سجل تحديثات (Timeline) لكل شكوى.
- البيانات محفوظة محلياً (localStorage) لأغراض العرض والتجربة.

## رفع المنصة على الإنترنت (النشر)

المشروع جاهز للنشر كموقع ثابت. تم إعداد الملفات التالية:

- **netlify.toml** — للنشر على [Netlify](https://netlify.com)
- **vercel.json** — للنشر على [Vercel](https://vercel.com)
- **.gitignore** — لاستبعاد الملفات غير الضرورية عند استخدام Git

### الطريقة الأسرع (Netlify — سحب وإفلات)

1. ادخل إلى [app.netlify.com](https://app.netlify.com) وسجّل الدخول.
2. اضغط **Add new site** → **Deploy manually**.
3. اسحب مجلد المشروع بالكامل (أو ضعه في منطقة السحب) وارفعه.
4. بعد انتهاء الرفع ستظهر لك رابط مثل: `https://اسم-عشوائي.netlify.app` — هذا هو رابط المنصة على الإنترنت.

### النشر عبر GitHub ثم Netlify أو Vercel

1. أنشئ مستودعاً جديداً على [GitHub](https://github.com/new).
2. من مجلد المشروع على جهازك نفّذ (استبدل `YOUR_USERNAME` و`REPO_NAME` باسمك واسم المستودع):

   ```bash
   git add .
   git commit -m "منصة صوت الموظف - جاهزة للنشر"
   git branch -M main
   git remote add origin https://github.com/YOUR_USERNAME/REPO_NAME.git
   git push -u origin main
   ```

3. في **Netlify:** Add new site → Import from Git → اختر المستودع → Deploy.
4. في **Vercel:** New Project → استورد المستودع → Deploy.

## ملاحظة

هذه واجهة أمامية فقط. للاستخدام الفعلي في المؤسسة يُنصح بربطها بخادم وقاعدة بيانات وحماية الصفحات الخاصة (لوحة المتابعة) بمصادقة حقيقية. بيانات الشكاوى حالياً تُحفظ في متصفح الزائر (localStorage) ولا تظهر لجميع المستخدمين بشكل موحّد إلا بعد إضافة خادم خلفي وقاعدة بيانات.
